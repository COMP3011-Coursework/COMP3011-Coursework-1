
## **Human** _2026-03-17 17:51 UTC_

why database url must have psycopg?

---

## **Assistant** _2026-03-17 17:51 UTC_

SQLAlchemy uses a URL scheme of `dialect+driver://...`. The `+psycopg` part tells SQLAlchemy which Python driver to use to connect to Postgres.

- `postgresql+psycopg` -> uses **psycopg v3** (the modern async-capable driver)
- `postgresql+psycopg2` -> uses **psycopg2** (older, sync-only)
- `postgresql://` alone -> SQLAlchemy picks a default driver, which may not be installed

Your backend uses `create_async_engine` (async SQLAlchemy), which requires psycopg v3 (`psycopg` package). Psycopg2 doesn't support async, so the `+psycopg` suffix is mandatory here.

The Render external URL uses the standard `postgresql://` scheme because it's meant for generic tools like `psql`/`pg_restore` --- you need to change the scheme when using it with SQLAlchemy.
