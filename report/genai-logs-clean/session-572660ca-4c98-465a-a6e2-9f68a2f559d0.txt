In schemas, what is this: Body_login_api_v1_auth_login_post

---

## **Assistant** _2026-03-17 01:25 UTC_

That's FastAPI's auto-generated OpenAPI schema name for the form body of your login endpoint. It's constructed as:

```
Body_{function_name}_{path_with_underscores}_{method}
Body_  login  _api_v1_auth_login_  post
```

It appears because `/auth/login` uses `OAuth2PasswordRequestForm` via `Depends()`, which is a **form-encoded** body (not JSON). FastAPI can't give it a clean model name since `OAuth2PasswordRequestForm` is a dependency, not a Pydantic model you defined --- so it generates that ugly name automatically.

It's harmless and expected. The only way to suppress it would be to replace `OAuth2PasswordRequestForm` with a custom Pydantic model and `Content-Type: application/json`, but that would break OAuth2 spec compatibility (clients expect `application/x-www-form-urlencoded` for the token endpoint). Leave it as-is.
