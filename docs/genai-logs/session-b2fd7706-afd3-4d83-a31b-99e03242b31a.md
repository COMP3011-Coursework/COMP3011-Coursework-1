## **Assistant** _2026-03-17 21:04 UTC_

`uvicorn.error` is a confusing name — it's actually uvicorn's main logger for **all** severities (INFO, WARNING, ERROR), not just errors. The name is historical.

FastAPI doesn't have its own separate logger. The convention in FastAPI apps is to use `logging.getLogger("app")` or similar, but those need manual setup. `uvicorn.error` works out of the box because uvicorn already configures it with handlers and a formatter.

If you want a cleaner name, the alternative is to keep `logging.getLogger(__name__)` (which gives `scripts.seed`) and add it to uvicorn's log config — but that's more boilerplate for the same result.

`uvicorn.error` is what FastAPI internally uses too (e.g. its startup "Application startup complete." message). It's the right choice here.
